# Example: Architecture Review

This example shows how to use GitHub Copilot + NotebookLM MCP to conduct an architecture review grounded in requirements, constraints, and standards stored in NotebookLM.

## Scenario

A team is reviewing the architecture for a new Azure-hosted data platform before go-live. Requirements, security constraints, and Well-Architected principles are stored in NotebookLM.

## Files

- [prompt.md](prompt.md) — The Copilot Chat prompt used
- [expected-output.md](expected-output.md) — Example of the expected output

## What this demonstrates

- Using NotebookLM to retrieve requirements before reviewing an architecture
- Generating a structured architecture review with findings and recommendations
- Grounding recommendations in source documents
