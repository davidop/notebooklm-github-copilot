# Governance

This page summarises the governance framework for enterprise adoption of the NotebookLM + GitHub Copilot integration.

## Governance documents

All governance documents are in the `governance/` folder:

| Document | Description |
|----------|-------------|
| Operating model | Roles, responsibilities, and review cadences |
| Acceptable use policy | What is and is not permitted |
| Data classification guide | How to classify sources before upload |
| Incident response | How to respond to data or security incidents |

Browse the full [governance/ folder](../governance/).

## Adoption maturity model

The repository includes a 5-level adoption maturity model (Level 0–4):

- **Level 0:** Ad hoc local experiments
- **Level 1:** Individual developer setup
- **Level 2:** Team-shared prompts and recipes
- **Level 3:** Governed enterprise rollout
- **Level 4:** Evaluated and continuously improved workflows

For the full model with criteria and exit conditions, see [docs/adoption-maturity-model.md](../docs/adoption-maturity-model.md).
