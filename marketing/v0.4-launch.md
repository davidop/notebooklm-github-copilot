An open-source toolkit that connects GitHub Copilot with Google NotebookLM via the Model Context Protocol (MCP). Copilot can query your private notebooks and return source-grounded answers with citations — directly in VS Code.

- JSON Schema definitions for structured AI outputs
- Adoption maturity model (5 levels)
- GitHub Pages documentation site

Works with VS Code + GitHub Copilot, Cursor, and OpenCode. Uses browser automation under the hood (notebooklm-mcp). Not affiliated with Google, GitHub, or Microsoft.

## GitHub release announcement

**v0.4.0 — Adoption-ready product experience**

This release evolves the toolkit from an enterprise reference kit into a demo-ready, easy-to-configure product that teams can try and adopt in minutes.

### Added

- **Setup wizard** (`npm run setup:wizard`) — Interactive CLI that generates MCP configs for VS Code, OpenCode, Cursor and generic clients
- **Client config generator** (`npm run setup:vscode`, `setup:opencode`, `setup:cursor`) — Non-interactive config generation
- **Doctor command** (`npm run doctor`) — Local readiness checker for Node.js, npm, client configs, package scripts and Chrome
- **GitHub Issues recipes** — 5 prompt-driven workflows: create issues, triage issues, generate PR descriptions, generate release notes, convert ADRs to issues
- **Azure DevOps recipes** — 5 prompt-driven workflows: create work items from meeting notes, epics/features/user stories, acceptance criteria, sprint planning, delivery risk review
- **Demo kit** — 2-min, 5-min and 15-min demo scripts, sample prompts, sample outputs and recording checklist
- **Sample sources** — 6 fictional demo documents for safe NotebookLM demos
- **Output schemas** — JSON Schema definitions for ADRs, architecture reviews, backlog items, presales proposals and risk registers, with example JSON files
- **Output format prompt packs** — 5 structured prompt packs for JSON and Markdown outputs
- **Adoption maturity model** — 5-level framework from ad hoc experiments to evaluated enterprise workflows
- **Client configuration matrix** — Comparison table for all supported MCP clients
- **GitHub Pages documentation site** — Static Markdown site in `site/`
- **SVG diagram assets** — Demo flow, client matrix, enterprise adoption flow
- **Release notes helper** (`npm run release:notes`) — Generates release notes from CHANGELOG.md
- **Improved validation** — `check:schemas`, `check:site`, updated `check:repo`

### Changed

- `package.json` version bumped to 0.4.0
- `check:repo` now runs `check:schemas` and `check:site`
- `check-recipes.mjs` now also scans `integrations/` directory
- `check-prompts.mjs` now handles output-format prompt packs with appropriate section checks
